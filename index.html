About

The OpenSHR is a shared health record (SHR) implementation that allows clinical data to be stored in a central location within a health information exchange (HIE). This enables continuity of care between multiple healthcare facilities by allowing facilities to submit and queyr clinical documents from a central location.

OpenSHR is standards-based and implements the IHE XDS.b profile for receiving and retrieving clinical documents about a particular patient.

This project is developed and maintained by Jembi Health Systems along with other partners for the OpenHIE project.

OpenSHR Features

Standards support: XDS.b

The OpenSHR support IHE's XDS.b standard for submitting and querying clincial documents. This enables a host of other systems to integrate with the OpenSHR seamlessly.

Discrete data storage

OpenSHR is able to deconstruct CDA documents that it receives so that data maybe be stored discretely in its internal data model.

A few core CDA documents are supported such as APHP, APS and IC, however, it is easy to add support for additional CDA document by wriiting content handler modules.

On demand document support

As the OpenSHR stores data discretely, it is also able to generate summary documents from a patient's entire clinical record. This enables better continuity of care.

The OpenSHR can generate summary documents such as IHE's XDS-MS specification. This makes use of the on-demand documents options of the XDS.b profile.

Get involved

If you want to get involved in the development of the OpenSHR then find the repositories on github. There you may submit issues if have any features request or find any bugs. You may also submit pull requests for changes that you'd like to see and the maintainers will review them.

The OpenSHR is used in the OpenHIE project as a reference implementation of a Shared Health Record (SHR). Join the SHR community or the OpenHIE initiative to participate.
